# v2.7.0 — Case S 추가 (Concept Art → Sprite Sheet)

**날짜**: 2026-06-06
**버전 범프**: 2.6.0 → 2.7.0 (마이너 — 새 평가 케이스 추가)

---

## 변경 동기

게임/웹 데모에 사용할 수 있는 **배경 투명 캐릭터 스프라이트 시트**를 컨셉아트로부터 자동 생성하는 워크플로우가 새롭게 요구됨.

- 기존 5개 케이스(A/B/C/D/W)는 모두 **벡터·HTML·디자인 시스템** 영역으로, 래스터/픽셀 자산 생성과 평가 기준이 없었음
- `pencil-design` 스킬의 이미지 생성 능력(`gemini_provider.edit()` + `comfyui_provider`)은 이미 존재했지만, **품질 평가축과 워크플로우가 미정**
- Case S는 컨셉아트 → 캐릭터 분리 → 다중 포즈 → 배경 투명 → 시트 패킹의 풀 파이프라인을 3축 100점으로 표준화

**최초 적용 대상**: `image/sprite/악단컨셉.png` (픽셀 음악단, 19명 캐릭터, 24×32px 픽셀 아트)

---

## 신규 케이스: Case S (Sprite)

| 축 | 이름 | 배점 |
|---|------|------|
| S1 | 캐릭터 충실도 (Character Fidelity) | 35 |
| S2 | 애니메이션 품질 (Animation Quality) | 35 |
| S3 | 공학적 활용성 (Engineering Usability) | 30 |

**대상 파일**: `design/sprite/output/_master/index.json` (PACKED MASTER 진입점)
**원본 보존 위치**: `image/sprite/` (원본 + 크롭 + 팔레트 + raw 생성물)

### 트리거 문구
- "악단컨셉 스프라이트 만들어줘"
- "캐릭터별 스프라이트 시트 생성"
- "컨셉아트에서 캐릭터 분리해줘"
- "배경 투명 스프라이트 추출"

### 핵심 워크플로우 (5 Phase)
1. **researching**: 컨셉아트 비전 분석 → 캐릭터 슬롯 추출, 글로벌 팔레트, 스타일 키워드
2. **designing 2a**: Gemini `edit()` 다중 포즈 생성 (시드 고정, 256×256 캔버스)
3. **designing 2b**: 후처리 — BiRefNet 매팅 → Unfake-Pixels 그리드 양자화 → 팔레트 강제 스냅 → 알파 이진화
4. **designing 2c**: 시트 조립 + Aseprite Hash JSON (캐릭터·액션별)
5. **designing 2d**: PACKED MASTER 시트 + `_master/index.json` (Phaser/Godot/Unity 즉시 로드)

### 산출물 디렉토리 구조
```
image/sprite/                         # 원본·중간 산출물
  악단컨셉.png                          # 원본 (보존)
  crops/{slug}.png                      # 캐릭터별 크롭 (edit() 입력)
  palette.json                          # 글로벌 팔레트
  raw/{date}-{slug}-{pose}-f{N}.png     # Gemini 원본 (256+px)

design/sprite/                        # 평가 대상 산출물
  sample/{NN}-{slug}.json               # Aseprite Hash 메타 (개별)
  output/
    {slug}/
      idle.png  play.png                # 액션별 시트
      sheet.json                        # 통합 메타
    _master/
      orchestra-master.png              # PACKED 마스터
      index.json                        # slug → 시트 매핑
```

### 캐릭터 slug 규칙 (악단컨셉 19개)
`vocal-{1..4}` / `piano, violin, viola, cello, contrabass, flute, clarinet, oboe, trumpet, trombone, horn, tuba, drum, guitar, harp`

## 변경 내역

### Layer 1 — Knowledge (design-craft.md)
- 1절 Case 표에 **Case S 행** 추가
- **4-3절 Case S: Concept Art → Sprite Sheet** 신규 섹션 (S1/S2/S3 기준·판정법)
- 7절 파이프라인 평가에 **S→W / S→B / A→S / S→B→W** 항목 추가
- 8절 평가 절차에 Case S 트리거·컨텍스트 수집 규칙 추가

### Layer 2 — Agents (design-evaluator.md)
- 역할 정의: 입력 항목에 "컨셉아트 + 스프라이트 시트(PNG/JSON)" 추가, 출력을 `A/B/C/D/S/W`로 확장
- 평가 기준: **Case S 3축 명세** 추가
- 결과 출력 형식: `Case {A|B|C|D|S|W}`로 확장
- 파이프라인 감지: `S→W (×1.3), S→B (×1.2), A→S (×1.2), S→B→W (×1.5)` 추가
- Recording 단계: 로그 파일명에 `case{A|B|C|D|S|W}` 반영

### Layer 3 — Engine (design-journey.md)
- 상태 다이어그램: Case A/B/C/D/**S**/W 유형 판별 추가
- researching/designing 상태 설명에 Case S 행동 추가
- **Case S 경로** 추가: `researching(컨셉아트 분석) → designing(Gemini edit + 후처리 + 시트 조립)`
- **Pipeline S→W**, **Pipeline A→S**, **Pipeline S→B→W** 경로 신규 추가
- 상태별 담당 표에 Case S 행동 반영

## 의존 패키지 (Case S 실행 시)

`requirements-sprite.txt` (별도 분리 — 기본 의존성 영향 없음):
- `Pillow` (≥10) — 이미지 양자화, 알파 처리
- `colorthief` — 글로벌 팔레트 추출
- `numpy` — 픽셀 그리드 분석
- `scikit-image` — SSIM, ΔE2000 측정
- `transformers` + `torch` — BiRefNet 모델 (~1GB)
- `opencv-python` — HSV 변환, 그린키 폴백

폴백 모드: BiRefNet 미설치 시 HSV 그린키 제거(`opencv-python`만) — 점수 천장이 22점으로 제한됨.
