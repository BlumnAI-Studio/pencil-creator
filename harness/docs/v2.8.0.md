# v2.8.0 — sprite-animator 전문가 도입 (캐릭터 컨셉아트 → 일관성 스프라이트 애니메이션)

**날짜**: 2026-06-12
**버전 범프**: 2.7.0 → 2.8.0 (마이너 — 첫 도메인 전문가 에이전트 추가)

---

## 변경 동기

v2.7.0의 Case S(컨셉아트→스프라이트 시트)가 첫 실행에서 평균 77/100 (B등급)을 받았고, sample14에서 S→W 파이프라인까지 실증됐다. 그러나:

1. **고품질 자산 제작은 단순 채점만으로 충분하지 않다** — 박스 좌표 정밀화, 캐릭터별 desc 작성, multi-frame reference chain, 머리 중복 fix 같은 **현장 의사결정**이 필요하다.
2. 6/8~6/12 동안 누적된 픽셀 음악단(19명, idle+play 152프레임) + 픽셀 댄스단(36명, dance 144프레임) 작업에서 **재현 가능한 성공·실패·교정 패턴**이 충분히 쌓였다.
3. 향후 다른 컨셉아트(전사단·요리사단 등)에 동일 파이프라인을 재사용하려면 **방법론을 명문화한 전문가**가 필요하다.

→ Case S 영역을 전담하는 **수행 + 자체 진단 일체형 전문가** sprite-animator를 도입한다.

## 신규 지식: sprite-animation-craft.md

| 위치 | `harness/knowledge/sprite-animation-craft.md` |
|------|----------------------------------------------|
| 분량 | 12개 섹션 |
| 출처 | 2026-06-06 ~ 2026-06-12 픽셀 음악단·댄스단 실제 작업 |

### 핵심 정리 내용
- **5 Phase 파이프라인**: analyze → pilot → batch → fix-pass → integrate
- **간소(single-pose) vs 풀(multi-frame) 모드 분기**: 간소 산출물을 풀 모드의 reference로 재활용하는 패턴
- **박스 좌표 정밀화**: std 기반 행/열 검출 (균등 분할 실패 사례 + 댄스단 정밀 y좌표)
- **캐릭터별 desc**: 머리색·의상색·소품 명시가 인접 침범 방지 핵심
- **Clean Frame as Reference**: 머리 중복 fix에서 **동일 캐릭터의 검증된 frame을 reference로 재호출**하는 결정적 기법 (clarinet·tuba·hiphop-1 f3 사례에서 1차 강조 프롬프트로 실패한 frame이 2차에서 100% 해결)
- **CSS calc 함정** + **padding 비례 스케일**: 192×192 frame에서 sample14 통합 시 비율 유지
- **실패 패턴 카탈로그**: 8개 증상 × 원인 × 치료
- **비용·시간 가이드**: 실측 기준 ($0.039/호출, 36×4=$5.96/30분 등)

```
idle → prompted → Phase 1 analyze → ✋ → Phase 2 pilot → ✋
     → Phase 3 batch (background) → Phase 4 fix-pass → ✋
     → Phase 5 integrate → design-evaluating → recording → idle
```

3개의 결정점(✋)에서 사용자 확인:
- Phase 1: 박스 좌표 시각 검증
- Phase 2: 시범 결과 OK?
- Phase 4: 의심 frame 추가 발견?
